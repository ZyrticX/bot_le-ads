## ⚠️ חשוב מאוד! פעולות שאתה חייב לעשות

### 🔴 עדכון שדות ב-Airtable (הכרחי!)

כדי שהבוט יעבוד נכון, **חובה** לעדכן את האופציות בשדות ה-Single Select ב-Airtable:

#### טבלת Deals:

**שדה Status:**
```
הישן: פעיל, הושלם, בוטל
החדש: Active, Completed, Cancelled
```

**שדה Deal Type:**
```
הישן: קנייה, מכירה
החדש: Buy, Sell
```

#### טבלת Clients:

**שדה Type:**
```
הישן: לקוח, ספק
החדש: Client, Supplier
```

#### טבלת Payments:

**שדה Type:**
```
הישן: התקבל, שולם
החדש: Received, Paid
```

---

## 📋 איך לעדכן שדה ב-Airtable?

1. פתח את ה-Base ב-Airtable
2. לחץ על כותרת השדה (למשל "Status")
3. לחץ "Customize field type"
4. תחת "Options" - **מחק את האופציות בעברית**
5. הוסף את האופציות החדשות באנגלית
6. לחץ "Save"

---

## 🧪 בדיקת התקינות

אחרי שעדכנת את Airtable, תבדוק:

### בדיקה 1: יצירת עסקה
```
1. שלח הודעה בטלגרם: "20 leads Italy"
2. Reply עם /newdeal
3. בחר "Sell"
4. שלח פרטי עסקה באנגלית
5. בדוק ב-Airtable שה-Status הוא "Active"
```

### בדיקה 2: הוספת לקוח
```
1. שלח /newclient
2. בחר "Client"
3. הזן שם
4. בדוק ב-Airtable שה-Type הוא "Client"
```

### בדיקה 3: סטטיסטיקות
```
1. שלח /stats
2. כל הטקסט צריך להיות באנגלית
```

## ❓ שאלות נפוצות

**ש: אם יש לי כבר נתונים בעברית ב-Airtable, מה קורה?**  
ת: תצטרך לעדכן אותם ידנית או ליצור Base חדש. ראה הסבר מפורט ב-IMPORTANT_CHANGES.md

**ש: האם שמות הטבלאות והשדות השתנו?**  
ת: לא! שמות הטבלאות (Deals, Clients, Payments) והשדות (Client, Supplier, etc.) היו כבר באנגלית.  
רק הערכים בשדות Single Select השתנו.

**ש: למה זה חשוב לעדכן את Airtable?**  
ת: כי הבוט עכשיו שומר "Active" ולא "פעיל". אם האופציות ב-Airtable עדיין בעברית, זה לא יתאים.

**ש: האם הפקודות השתנו?**  
ת: לא! הפקודות נשארו אותו דבר: /start, /newdeal, /newclient, וכו'.
